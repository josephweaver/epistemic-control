Implementation Execution Order
1. Added versioned schema tables (`workflow_dependency_steps`, `workflow_dependency_work_items`) to SQLite bootstrap and schema-existence checks.
2. Added store records, list methods, and conflict-aware insert/idempotent behavior for persisted dependency steps and work-item memberships in `internal/persistence/store.go`.
3. Persisted dependency plan step facts in `CreateWorkflowDependencyPlan` before JSON state writes.
4. Persisted dependency membership facts in `RecordCompiledWorkItemMembership` before JSON state writes.
5. Added focused tests in `internal/persistence/store_test.go` and `cmd/controller/workflow_dependency_store_test.go` for deterministic order, rejection, idempotency, and schema creation.
6. Updated OS doc status and `PROJECT_STATE.md`.

Behavior and Risk Notes
- The slice preserves existing state flow semantics and keeps dependency execution status in `submission_context_json` for this phase.
- Foreign-key enforcement to `work_items` for `workflow_dependency_work_items` was intentionally omitted to preserve existing controller tests and current dependency compilation flow where memberships are persisted before durable work-item rows are guaranteed in all contexts.
- The primary key constraints remain `(run_id, work_item_id)` and `(run_id, step_index)` on dependency steps, with explicit deterministic ordering via SQL `ORDER BY`.

Validation
- `go test ./internal/persistence -count=1` ✅
- `go test ./cmd/controller -count=1` ✅

Edge Cases Covered
- Deterministic ordering of persisted steps via `ORDER BY stage_index, step_index`.
- Deterministic membership list order via `ORDER BY work_item_index, work_item_id`.
- Duplicate step plan insertions with identical content are idempotent.
- Duplicate membership inserts with identical content are idempotent.
- Constraint conflicts are surfaced for duplicate plan or work-item index conflicts.

Potential Ambiguity / Follow-up
- If team preference is to keep strict FK enforcement to `work_items`, `workflow_dependency_work_items` inserts and related call sites must be aligned so work-item rows are created before membership persistence in all paths.
