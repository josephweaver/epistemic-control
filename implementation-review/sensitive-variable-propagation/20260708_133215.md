## 1. Intended Design

Strategic Concept: `docs/concepts/sensitive-variable-propagation/README.md`.

Operational Slice: `docs/concepts/sensitive-variable-propagation/004-worker-secret-resolver-and-redactor.md`.

Objective: add worker-side primitives that resolve protected references into typed sensitive values only inside the worker execution boundary, and add an attempt-local redactor that scrubs exact materialized secrets from controlled strings or byte streams.

Explicit non-goals from the OS:
- No controller compilation changes.
- No Python-specific materialization.
- No Go work-item handler migration.
- No end-to-end smoke.
- No external secret manager providers.
- No artifact-file scanning.

Acceptance criteria reviewed:
- `worker_env` protected references resolve from the worker environment.
- Missing worker env and unsupported providers return sanitized errors.
- `SensitiveValue` default string, JSON, and error formatting is redacted.
- Plaintext access requires an explicit method call.
- Attempt-local redactor replaces exact literal materialized secrets, handles multiple secrets deterministically, and does not persist its registry.
- Worker tests pass.

## 2. EC Object / Transition Map

Sentence 1:

`ProtectedValueRef` transitions from durable non-secret reference state to in-memory `SensitiveValue` state when a worker operation explicitly calls `ResolveProtectedValue`, by reading the named `worker_env` key, while preserving the invariant that default formatting and JSON rendering do not expose plaintext.

Objects:
- Persistent data object: none created by this slice.
- In-memory data objects: `ProtectedValueRef`, `SensitiveValue`.
- Config objects: worker process environment.
- External artifacts: environment variable named by the reference key.
- Before: reference contains provider, key, type, and redaction label.
- After: `SensitiveValue` holds plaintext in unexported fields and exposes `Plaintext()` for deliberate access.
- Failure states: unsupported provider, empty key, missing environment key, canceled context.
- Evidence: `cmd/worker/protected_value.go`; tests in `cmd/worker/protected_value_test.go`.

Sentence 2:

`Redactor` transitions from empty attempt-local registry to registered literal-secret registry when a materialized `SensitiveValue` or plaintext string is registered, by appending an in-memory entry, while preserving the invariant that replacement is literal, deterministic, and not persisted through JSON.

Objects:
- Persistent data object: none.
- In-memory data object: `Redactor.entries`.
- Config objects: none.
- External artifacts: controlled strings or byte streams passed to redaction methods.
- Before: no registered secrets.
- After: exact registered strings are replaced with labels.
- Failure states: nil redactor is a no-op; empty secret is ignored.
- Evidence: `cmd/worker/redactor.go`; tests in `cmd/worker/redactor_test.go`.

## 4. Transition-Order Review

### Protected reference resolution

Input object: `ProtectedValueRef`, an alias to `model.ExecutionEnvelopeProtectedReference`.

Precondition: caller invokes `ResolveProtectedValue(ctx, ref)` inside worker code. This slice does not automatically resolve every assignment.

Operation: `WorkerEnvProtectedValueResolver.ResolveProtectedValue` checks context cancellation, trims provider and key for validation, accepts only `worker_env`, looks up the key through `os.LookupEnv` or injected `LookupEnv`, and constructs `SensitiveValue`.

After state: returned `SensitiveValue` contains type, plaintext, and redaction label in unexported fields.

Invariant: unsupported provider errors do not echo the key; missing env errors use the redaction label rather than a plaintext value.

Evidence: `protected_value.go` and tests `TestWorkerEnvProtectedValueResolverResolvesWorkerEnv`, `TestWorkerEnvProtectedValueResolverMissingEnvReturnsSanitizedError`, and `TestWorkerEnvProtectedValueResolverUnsupportedProviderReturnsSanitizedError`.

### Sensitive value rendering

Input object: `SensitiveValue`.

Precondition: worker code has already resolved or constructed a value.

Operation: default formatting methods `String`, `GoString`, `Error`, and `MarshalJSON` return a redaction label. `Plaintext()` is the only method that returns the secret.

After state: accidental formatting surfaces contain labels, not plaintext.

Invariant: the plaintext field is unexported and does not appear through tested formatting or JSON.

Evidence: `TestSensitiveValueDefaultFormattingAndJSONAreRedacted`.

### Attempt-local redaction

Input object: `Redactor`.

Precondition: worker code registers a materialized secret for the current attempt.

Operation: `Register` or `RegisterPlaintext` appends a non-empty secret and label. `RedactString` copies entries, sorts longer secrets before shorter overlapping secrets, and applies `strings.ReplaceAll`. `RedactBytes` wraps the string redaction path.

After state: exact literal occurrences are replaced with labels.

Invariant: replacement treats the secret as text, not a regular expression; JSON marshaling of `Redactor` emits `{}`.

Evidence: `TestRedactorReplacesExactMaterializedSecretLiterally`, `TestRedactorHandlesMultipleSecretsDeterministically`, `TestRedactorRedactsByteStreams`, and `TestRedactorDoesNotPersistRegistry`.

## 6. Concrete Trace

Initial input:

```go
ProtectedValueRef{
    Type: "string",
    Provider: "worker_env",
    Key: "GOET_TEST_TOKEN",
    RedactionLabel: "${worker_env.GOET_TEST_TOKEN}",
}
```

Environment:

```text
GOET_TEST_TOKEN=plain-secret
```

Call sequence:
1. Worker code calls `WorkerEnvProtectedValueResolver{}.ResolveProtectedValue(ctx, ref)`.
2. Resolver accepts provider `worker_env`.
3. Resolver calls `os.LookupEnv("GOET_TEST_TOKEN")`.
4. Resolver returns `SensitiveValue{typ: "string", plaintext: "plain-secret", redactionLabel: "${worker_env.GOET_TEST_TOKEN}"}`.
5. `fmt.Sprint(value)` returns `${worker_env.GOET_TEST_TOKEN}`.
6. `value.Plaintext()` returns `plain-secret`.
7. `redactor.Register(value)` records the exact plaintext for this redactor instance.
8. `redactor.RedactString("stdout plain-secret")` returns `stdout ${worker_env.GOET_TEST_TOKEN}`.

Failed precondition trace:
- If `GOET_TEST_TOKEN` is absent, the resolver returns `protected value ${worker_env.GOET_TEST_TOKEN} is unavailable in worker environment`; no plaintext exists to leak.

## 8. Risks and Ambiguities

Risk: empty `worker_env` values resolve as `SensitiveValue` with empty plaintext.
- Location: `WorkerEnvProtectedValueResolver.ResolveProtectedValue`.
- Impact: future materialization code may need to decide whether empty secrets are valid.
- Action: defer unless a later slice defines stricter validation.

Risk: missing-env sanitized error includes the redaction label, which may include the environment key.
- Location: `protectedValueRedactionLabel`.
- Impact: environment key names are protected references, not plaintext secrets, according to the concept. If future policy treats key names as sensitive, error wording should change.
- Action: defer; current concept permits protected reference identity in controller snapshots and assignments.

Risk: redaction is exact only.
- Location: `RedactString` and `RedactBytes`.
- Impact: split, transformed, encoded, or hashed leaks are not caught.
- Action: accepted by the concept redaction contract.

## 10. Final EC Summary

### What I should understand before moving on

- `ProtectedValueRef` is still a non-secret reference object from the execution envelope.
- `WorkerEnvProtectedValueResolver` is the only real provider implemented in this slice.
- `SensitiveValue` can hold plaintext, but formatting and JSON return redaction labels.
- Plaintext access is visible in code review through `Plaintext()`.
- `Redactor` is per-instance, in-memory, exact, literal, and not persisted.
