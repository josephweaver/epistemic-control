## Intended Design

The slice objective is to implement GDAL-backed raster reprojection and grid alignment for categorical rasters. The accepted operations are `reproject_crs` and `align_to_grid`. The explicit acceptance criteria require tiny fixture alignment, default nearest-neighbor resampling, unsafe categorical resampling rejection unless explicitly allowed, `like_raster` target grids, source/target/nodata/GDAL metadata output, clear missing-CRS failure, incomplete-grid failure before writing, artifact-root path confinement, and default tests that do not require GDAL.

Out of scope remains pair counting, stacking, polygon cropping, continuous resampling policy beyond explicit opt-in, real CDL/Yan/Roy files, automatic CRS guessing, and filename-convention alignment.

## Object / Transition Map

`OperationRequest` transitions from decoded JSON to validated alignment request when the request has operation `align_to_grid` or `reproject_crs`, input `source_raster`, outputs `raster_tif` and `metadata_json`, and exactly one target mode. The operation is `ParseAlignmentRequest` in `internal/geospatial/align_gdal.go`. It preserves the invariant that categorical resampling is `nearest` unless `allow_unsafe_resampling` is true.

`source raster file` transitions from original grid to target grid when source CRS exists, target grid is complete, output paths are safe artifact-relative paths, and `gdalwarp` succeeds. The operation is `ExecuteRasterAlignment` -> `runGDALWarp`. It preserves the invariant that output artifacts are written under the artifact root derived from the response path.

`alignment metadata JSON` transitions from absent to written artifact after output raster metadata is read. The operation is `writeJSONFile` in `align_gdal.go`. It records source grid, target grid, output grid, resampling, nodata policy, and GDAL version.

## Transition Review

Request validation starts in `OperationRequest.Validate()`. For `align_to_grid` and `reproject_crs`, it calls `ParseAlignmentRequest`. The parser requires `source_raster`, output names `raster_tif` and `metadata_json`, band 1, and one target mode. It rejects mixed `like_raster` plus explicit grid fields, rejects incomplete explicit grids, rejects rotated/sheared target transforms, and rejects non-nearest resampling unless the request sets `allow_unsafe_resampling`.

Execution starts in `cmd/goet-geospatial/main.go` after envelope validation. The CLI passes `filepath.Dir(responsePath)` as the artifact root to `ExecuteRasterAlignment`. That function validates again, reads source metadata, fails if `CRSWKTPresent` is false, resolves either the explicit target grid or the `like_raster` metadata grid, and calls `artifactPath` for both output paths.

`runGDALWarp` constructs argv as a string slice: `gdalwarp -overwrite -of GTiff -t_srs <crs> -te <bounds> -ts <width> <height> -r <method> ...`. It does not shell-concatenate. When nodata is declared in the request or present on band 1, it passes that value as both `-srcnodata` and `-dstnodata`.

After `gdalwarp`, `ExecuteRasterAlignment` reads the output raster metadata, calls `gdalinfo --version`, writes the metadata JSON artifact, and returns an `OperationResult` with raster and metadata artifacts.

## Concrete Trace

Input request: `operation = "align_to_grid"`, `inputs.source_raster.path = source.tif`, `outputs.raster_tif = aligned/cdl.tif`, `options.target_crs = EPSG:5070`, `target_transform = [0,30,0,120,0,-30]`, `target_width = 4`, `target_height = 4`.

`run()` decodes JSON into `OperationRequest`, then `Validate()` calls `ParseAlignmentRequest`. The parser creates `AlignmentRequest` with default `Resampling = "nearest"`. `ExecuteRasterAlignment` reads `source.tif` metadata, verifies CRS exists, computes bounds `[0,0,120,120]` from the target grid, creates the artifact parent directory, and executes `gdalwarp`. It then reads `aligned/cdl.tif`, writes `aligned/cdl.metadata.json`, and the response JSON lists both artifacts.

If `source.tif` has no CRS, the transition stops after source metadata read with `source raster CRS is missing; refusing to guess`, and no raster artifact is written.

## Risks And Gaps

`like_raster` currently requires the CRS to resolve to an EPSG code because the compact `RasterMetadata` object records `EPSG` and `CRSWKTPresent`, not full WKT. This is acceptable for the tested CDL/Yan/Roy fixture path using `EPSG:5070`, but future non-EPSG WKT-only rasters need either explicit `target_crs` or a metadata extension.

The implementation supports north-up, non-rotated target transforms only. Rotated/sheared grids are rejected. This is a deliberate fail-closed choice because the OS emphasizes scientific correctness and pixel alignment.

`reproject_crs` uses the same target-grid execution path as `align_to_grid`; the current slice does not implement a target-CRS-only mode with inferred dimensions or resolution. The OS target modes require either a complete explicit grid or `like_raster`, so this matches the stated contract.

The full GDAL tests were run through WSL with GDAL 3.8.4, not through the Docker container script. The OS allowed GDAL tests in the GDAL container path and the user suggested WSL for GDAL testing; WSL was used.

## Final EC Summary

What I should understand before moving on:

- The alignment request is valid only with `source_raster`, `raster_tif`, `metadata_json`, and exactly one target mode.
- The default categorical resampling method is `nearest`.
- Unsafe resampling methods require `allow_unsafe_resampling: true`.
- Missing source CRS and incomplete target grids fail before output is written.
- Output metadata records source, target, output, resampling, nodata policy, and GDAL version.
