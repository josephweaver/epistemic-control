## 1) Intended Design

The slice objective is to add a narrow, read-only `raster_info` operation to `goet-geospatial`.
`OperationRequest` remains `map[string]InputSpec` for `inputs`, and the operation must:

- emit raster metadata records for every input, ordered lexicographically by input name,
- write metadata under `outputs["metadata_json"]`,
- register `raster_info` in production validation/dispatch,
- keep default tests runnable without GDAL while keeping GDAL-only behavior in `-tags gdal` tests and container smoke paths.

## 2) Transition Map

`OperationRequest` transitions from untrusted JSON bytes to an in-memory valid request when `APIVersion`, `Kind`, and supported operation pass validation (`internal/geospatial/validation.go`), while output-path safety invariants are preserved.

When operation is `raster_info`, each `inputs[name]` transitions from path string to ordered `RasterMetadata` through GDAL extraction (`internal/geospatial/raster_info_gdal.go`), then transitions again to a deterministic artifact and response summary written by `cmd/goet-geospatial/main.go`.

## 4) Review in Execution Order

1. Request parsing and validation:
   - `cmd/goet-geospatial/main.go` reads/parses JSON, then calls `request.Validate()`.
   - `internal/geospatial/validation.go` now accepts `raster_info`, requires at least one input and `metadata_json`.
2. Metadata extraction:
   - `CollectRasterMetadata` (in `raster_info_gdal.go`) sorts input names, calls GDAL (`gdalinfo -json`) per input, extracts driver/size/geoTransform/bands/CRS/WKT/EPSG, computes bounds from geoTransform, and returns records.
3. Response/artifact materialization:
   - `cmd/goet-geospatial/main.go` writes `metadata` artifact under `filepath.Join(filepath.Dir(*responsePath), outputs["metadata_json"])`, sets artifact metadata, and includes `"rasters"` in `summary`.
4. Result emission:
   - `OperationResult` is marshaled deterministic JSON with added artifact entry.

Failure behavior:
- Validation errors return immediately and prevent response/artifact writes.
- GDAL failures are returned by `CollectRasterMetadata`.
- Non-GDAL builds return a clear stub error from `raster_info_nogdal.go`.

## 6) Risks / Ambiguity

- GDAL dependency behavior is intentionally isolated behind `-tags gdal`; behavior is inferred from script and tests rather than local runtime.
- Container smoke ordering checks rely on string line ordering in emitted JSON; this is stable due `json.MarshalIndent` with ordered struct fields but still a mildly brittle assertion style.
- `raster_info` now requires map-based inputs and lexicographic order; this is explicitly aligned with the revised objective but incompatible with any previous array-based assumption.

## 7) Budget and Slice Fit

Scope matches OS-003 with only the listed geospatial/plugin and allowed container changes.
The implementation stayed mechanical and did not introduce alignment, counting, reprojection, or workflow orchestration changes.
