## Intended Design

This slice adds a new operation to `goet-geospatial`:

`stack_aligned_rasters` composes aligned single-band categorical inputs into one multi-band GeoTIFF.

Requested acceptance behavior:

- reject unsupported operations or malformed requests at validation time,
- require `field_id`/`crop_id`-style inputs with explicit `band` and `output_band`,
- require same CRS + transform + width + height alignment before writing,
- emit stacked raster + metadata artifacts,
- default `dtype` and `nodata` where not set,
- keep behavior compatible with existing non-GDAL tests by avoiding GDAL calls during validation.

## Object / Transition Map

- `OperationRequest` now includes `output_band` on `InputSpec` to encode destination band mappings.
- `ParseStackAlignedRequest` validates request-level constraints (operation, outputs, options, output_band shape) without calling GDAL.
- `ExecuteStackAlignedRasters` then:
  - reads input metadata with `collectSingleRasterMetadata`,
  - validates CRS and grid compatibility,
  - builds temp VRT inputs, then stacks with `gdalbuildvrt -separate`,
  - writes GeoTIFF with `gdal_translate`,
  - emits deterministic `StackAlignedMetadata`.
- CLI dispatch in `cmd/goet-geospatial/main.go` routes `stack_aligned_rasters` into execution.

## Transition Review

Validation now rejects stack requests with:

- missing input/output components,
- missing `output_band`,
- duplicate output bands,
- unsupported output dtype.

This is done in `ParseStackAlignedRequest`, which is invoked from `OperationRequest.Validate` under `OperationStackAligned`.

Execution validates only after parsing succeeds:

- ensures every input declares CRS and band exists,
- enforces each source dtype equals `UInt16` for this slice,
- enforces aligned raster grid (`GridsEqual`),
- writes `stacked_raster` and `metadata_json` artifacts.

Metadata includes source names, source bands, output band order, grid, dtype, nodata, and `gdal_version`.

## Risks and Ambiguities

1. The slice intentionally supports only `UInt16` output/input compatibility; mixed dtypes are rejected even if values could still be safely represented.
   This is documented as a conservative first path.
2. Output band numbers must be contiguous and start at 1. This matches current tests and avoids gaps.
3. Validation depends on GDAL metadata resolution; CRS missing or non-EPSG metadata remains unsupported and is rejected as specified by stricter safety behavior.
